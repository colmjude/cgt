Cross government tools

Capturing civil servant's user needs

David Mann, 24 March 2015

From our experience of working on the GOV.UK transition project, one of the most useful tools we had was Maslow. Maslow is a piece of software that helps us record and manage the user needs behind all the content that is published on GOV.UK.

As we are running a project to define and test the user needs of civil servants, we felt Maslow would be an invaluable tool for us. We can record not just descriptions of user needs but also supporting evidence to show why they are important and criteria for us to measure when a need is met.

On the GOV.UK transition project, Maslow made it easier for us to explain user needs to our colleagues across government, and provided a space where we could collaborate about the ways in which we could best meet those needs.

Since we first talked about Maslow a few years ago, other people have asked us how they could use it for their own projects. Dan Blundell recently wrote about getting Maslow up and running outside the GOV.UK stack.

We’ve been working on a fork of Maslow which is de-coupled from other GOV.UK components (like our single sign-on service), so that other teams and projects can use it. We’ve simplified how we store the needs data, using a PostgreSQL database instead of the Need API. We’ve also removed other dependencies on our Content API and the transition workflow processes.

In addition, we’ve started work on features to make the data we capture more flexible - for example, whilst capturing information about historic pageviews makes sense for a GOV.UK need, it’s not a useful measure for a civil service need.

We need to be able to map needs we record to existing services, for example Civil Service Learning. Also, we need to be able to categorise needs according to our research framework [link to post about the journey] and sprint themes [link here to Elisse’s picture].

We’ve changed the existing decision-making process for a need, so that we can now measure three kinds of decisions: whether a need is in scope, whether it has complete information, and whether it’s being met or not. Every decision is saved, so you can look back at a documented history of the need.

If you’re interested in using Maslow for your organisation, the code for our work-in-progress fork is on GitHub.

We’re going to continue iterating Maslow as we continue our discovery phase and we’ll report back on our changes in the future.
